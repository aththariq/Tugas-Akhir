\chapter{CONTOH ARTEFAK KANAL DATA TAMBAHAN}

Lampiran ini menyajikan contoh artefak pertukaran data yang relevan dengan
implementasi arsitektur sistem inspeksi kargo digital. Artefak yang ditampilkan
bukan dimaksudkan sebagai dokumentasi seluruh kode program, melainkan sebagai
contoh kanal data tambahan yang pernah digunakan untuk kebutuhan pelacakan
objek. Kanal ini berbeda dari jalur operasi utama inspeksi kontainer, yang
berjalan melalui antarmuka \texttt{/api/edge} untuk hasil OCR, pemindaian
kerusakan, pemindaian barang berisiko, dan klasifikasi muatan.

\section{Contoh Payload Hasil Pelacakan Tambahan dari \textit{Edge} ke Layanan API}

\begin{lstlisting}[caption={Contoh payload kanal pelacakan tambahan dari komponen \textit{edge} ke layanan API}, label={lst:tracking-payload}]

  "camera_id": "gate-camera-1",
  "device_id": "edge-device-001",
  "timestamp": "2025-12-06T10:30:00Z",
  "frame_number": 145,
  "model_key": "yolo",
  "trackings": [
    {
      "tracking_id": 1,
      "class_name": "truck",
      "confidence": 0.95,
      "box": {
        "x1": 100,
        "y1": 200,
        "x2": 500,
        "y2": 600
      },
      "metadata": {
        "direction": "entering",
        "speed": 5.2,
        "ocr_text": "MSCU1234567",
        "ocr_confidence": 0.92
      }
    }
  ]
}
\end{lstlisting}

Listing~\ref{lst:tracking-payload} menunjukkan contoh kanal data tambahan yang
pernah digunakan untuk mengirim informasi pelacakan objek dalam bentuk data
terstruktur. Payload ini tidak mewakili jalur operasi utama inspeksi kontainer,
melainkan kanal pendukung yang terpisah dari alur utama \texttt{/api/edge}.
Lampiran ini dipertahankan untuk menunjukkan bahwa sistem pernah memiliki
eksperimen kanal data tambahan, tetapi pembahasan inti penelitian tetap
berfokus pada jalur inspeksi utama dan pemisahan antara data terstruktur dengan
video langsung.
